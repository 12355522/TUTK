前言

AV Module 係為 IOTC 平台中專為了傳送影音資料所設計，可應用的範圍包含 Set Top Box、IPTV、IP Camera等。

在IP Camera應用領域中，我們推出了一套以 AV Module 實作的客戶端，該客戶端可完整支援依 AV Module 實作的設備。然而設備所提供的功能十分多樣，為使客戶端與設備端之間的功能能夠同步化，遂制定「服務類別」 ( Service Type ) 以區別設備裝置是否有無支援某特定功能。

使用說明

「服務類別」 (Service Type) 的定義是由設備端 (AV Server) 進行定義，並透過 avServerStart() 函式 (在客戶端是藉由avClientStart()) 來進行傳遞，我們先來看這個函式的定義：

int  avServStart(int nIOTCSessionID, char *szViewAccount, char *szViewPassword, unsigned long nTimeout, unsigned long nServType, unsigned char nIOTCChannelID);

int  avClientStart(int nIOTCSessionID, char *szViewAccount, char *szViewPassword, unsigned long nTimeout, unsigned long *pnServType, unsigned char nIOTCChannelID);

由上述的二個函式定義可看出是透過一個「unsigned long」的參數進行傳遞，而「unsigned long」有4個bytes，也就是有32個bits，

如果以二進位制表示，可以看做：

「00000000 00000000 00000000 00000000」

將每個位元視為一個功能的標記，共可得到32個功能標記。

若一個已定義的位元其值如為「0」，則表示該功能為有效的；反之，其值若為「1」，則表示該功能是失效的。舉例來說，Audio In 功能是定義在索引位置為0的位元上，倘若位元索引位置0的值為「0 」則表示 Audio In 功能是有效的。再舉個例子，Pan/Tilt 功能是定義在索引位置為2的位元上，倘若位元索引位為2的值為「1」則表示該設備不支援Pan/Tilt 功能。
